# Accessibility Plan

## Goal

The Campus Life Tracker should be accessible to all students, including those who use screen readers, keyboard navigation, or need high contrast text.

## Approach

- Use semantic HTML elements like `button`, `label`, `form`, `table`, and `nav`
- Ensure every form input has a visible label
- Add `aria-label` or `aria-describedby` where additional context is needed
- Make all controls keyboard accessible
- Provide clear focus states for links, buttons, and form fields
- Use high contrast colors for text and background
- Keep the layout responsive for different screen sizes
- Write clear text for error messages and feedback
- Test with keyboard-only navigation and a screen reader

## Notes

This plan emphasizes structure, clarity, and keyboard friendliness so the app is usable by more students.
